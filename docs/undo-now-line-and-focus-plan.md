# Undo Callout: Now Line Visibility & Focus-on-Undo Plan

## Plan

Verify and refine the Text History (Undo) callout in [src/app/ui/dialogs/text_history.rs](src/app/ui/dialogs/text_history.rs) and the underlying undo/redo flow in [src/app/text_history.rs](src/app/text_history.rs) so that: (1) `render_now_line` is suppressed when no entry is `undone` (i.e. the Now line sits at the very top of history with nothing to redo above it) — confirm by inspecting `newest_applied_index`/`now_line_belongs_above` against a fixture where every row has `undone == false`, then short-circuit `render_timeline_rows` (and the `render_by_file` per-group case) to skip emitting the line when `chronological.iter().all(|r| !r.undone)`; (2) clicking any timeline row already routes through `app.undo_text_history_entry`/`redo_text_history_entry`, so confirm via a manual run that the Now line repositions so undone rows render above it as redo candidates with `UNDONE_OPACITY` styling, and add a regression check that walks the chronological list after a click to assert the boundary moved to the clicked entry; (3) on undo, the editor focus/selection should jump to the change site of the *most recently performed* undo in that batch — extend the undo path to return the affected `BufferId` plus a cursor anchor derived from the reversed `PieceHistoryEdit` (insert site for an inserted-then-undone edit, deletion site for a deleted-then-undone edit), then have `show_text_history_window` request a buffer activation + cursor move on the returned anchor; (4) add an affordance in the search/undo callout header (a small toggle button next to the tabs in `render_tabs`, persisted via `widget_ids::global("text_history.follow_undo")` like `read_active_tab`) labelled e.g. "Follow undo" defaulting to on, and gate the focus-jump on that flag so power users can undo many steps without losing their place. Validation: build the app, open a buffer, perform several edits, open the callout, undo to top → confirm Now line disappears; redo partway → confirm Now line reappears between applied and undone rows; toggle "Follow undo" off → confirm cursor stays put; toggle on → confirm cursor lands at the last-undone change.
